CHUNK STRUCTURE:
	Chunk ID:   | 8 bytes (char)    | A string identifying the chunk content.
	Chunk size: | 8 bytes (uint64)  | The size of the chunk data in bytes.
	Chunk data: | any size and type | The chunk data.

NUMAV RESULT FORMAT SIGNATURE
	8 bytes (char): the .nmvr format ID ("numavrst")

CHUNK: simulation type
	Chunk ID:
		"sim_type"
	Chunk size:
		40 bytes
	Chunk data:
		8 bytes (char): modeled phenomenon ("acoustic")
		8 bytes (char): numerical method ("fem_____")
		8 bytes (char): simulation domain ("freq_dom")
		8 bytes (char): dimension of the elements ("3_dimens")
		8 bytes (char): order of elements ("1st_ord_" or "2nd_ord_")

CHUNK: simulated frequency steps
	Chunk ID:
		"freq_stp"
	Chunk size:
		Dynamic
	Chunk data:
		any number of bytes (double): sequence of frequency steps in Hz

CHUNK: node index to coordinates
	Chunk ID:
		"ni2coord"
	Chunk size:
	 	Dynamic
	Chunk data:
		any number of bytes (double): sequence of coordinates of each node

CHUNK: surface element index to node index
	Chunk ID:
	 	"sei_2_ni"
	Chunk size:
	 	Dynamic
	Chunk data:
	 	any number of bytes (uint64): the indices of nodes that compose each
		surface element

CHUNK: volume element index to node index
	Chunk ID:
		"vei_2_ni"
	Chunk size:
	 	Dynamic
	Chunk data:
		any number of bytes (uint64): the indices of nodes that compose each
		volume element

CHUNK: complex amplitude of sound pressure solution
	Chunk ID:
	 	"cpx_pres"
	Chunk size:
	 	Dynamic
	Chunk data:
		any number of bytes (complex double): the matrix of complex amplitudes
		of sound pressure, with nodes as rows and frequencies as columns, in
		column major form
